# 글로벌 작가 온보딩 — 모바일 친화 리뉴얼 & 아이콘/컴포넌트 개선안

작성일: 2025-12-23  
목표: **모바일에서 “불편/어색/잘림”을 제거하고**, 아이콘/컴포넌트 규칙을 통일해 **전문성·신뢰감·전환율**을 끌어올립니다.

---

## 1) 문제 정의(현재 모바일에서 어색한 이유)

- **레이아웃 과밀**: 3~4열/고정 폭 요소(표/배지/아이콘)가 작은 화면에서 압축되며 텍스트가 잘리거나 줄바꿈이 불안정.
- **CTA 접근성 부족**: 학습(step) 페이지에서 “이전/다음”이 하단 고정이 아니라 스크롤 하단에 있어 엄지 영역에서 멀어짐.
- **아이콘 스타일 혼재**: 이모지(📚/🎯/🔒) + 텍스트 심볼(→/✓/←) 혼합 → 브랜드 톤이 약하고 화면마다 인상이 달라짐.
- **정보 밀도 편차**: 같은 “카드”라도 패딩/텍스트 크기/라인 처리 방식이 제각각이라 시각적 피로 증가.

---

## 2) 모바일 UX 원칙(이 프로젝트 기준)

- **원칙 A — 하단 CTA는 항상 엄지 영역**: 모바일에서 핵심 액션(다음/완료)은 하단 고정 + safe-area 대응.
- **원칙 B — 텍스트는 2줄까지만 보장**: 설명문은 기본 `line-clamp-2`로 과도한 세로 길이/피로를 제한.
- **원칙 C — 표는 모바일에서 “카드형 행”으로 변환**: 가로 스크롤 표는 정보 습득이 느림 → 행을 카드로 재구성.
- **원칙 D — 아이콘은 한 스타일(선형 SVG)**: UI용 아이콘은 동일 stroke/크기로 통일, 이모지는 “콘텐츠 강조”에만 제한 사용.
- **원칙 E — 터치 타겟 최소 44px**: 버튼/칩/토글은 최소 높이 확보.

## 3) 컴포넌트 개선 설계(구체)

### 3-1. `ResponsiveTable` 도입(모바일 표 최적화)
- **Desktop/Tablet(sm 이상)**: 기본 `<table>` 렌더 (min-width 부여 + overflow-x)
- **Mobile(sm 미만)**: 각 행을 카드로 렌더  
  - 헤더: “수수료 항목: idus 수수료(동일)”  
  - 본문: “일반/프리미엄/새싹/국가공인” 값을 2열 그리드로 표시

효과:
- 작은 화면에서 “표 잘림/축소” 문제 제거
- 스크롤 없이 핵심 비교가 가능

### 3-2. 학습 Step 페이지 하단 CTA 고정 + safe-area
- 모바일에서는 하단에 고정 바(`fixed bottom-0`)를 두고 `pb-[env(safe-area-inset-bottom)]` 적용
- 본문은 `pb-28` 등으로 하단 바만큼 여백을 확보해 콘텐츠가 가려지지 않게 처리

효과:
- 다음/이전 액션이 항상 보임 → 이탈 감소
- 한 손 조작(엄지) UX 개선

### 3-3. 카드/텍스트 스케일 통일
- 카드 내부 패딩: `p-4 sm:p-6` 기본
- 설명 텍스트: `line-clamp-2` 기본(필요 시 예외만 허용)
- 아이콘 박스: `w-12 h-12 (모바일)`, `w-16 h-16 (데스크톱)`

## 4) 아이콘 시스템 개선(설계/대안)

### 4-1. 현재 적용(즉시 적용 가능한 방식)
- 외부 설치 없이 **내장 SVG 아이콘 세트**(`src/components/ui/icons.tsx`)로 UI 아이콘 통일
- 적용 대상:
  - 뒤로가기/다음/이전/체크/잠금/플러스(FAQ 토글)/외부 링크 화살표

장점:
- 빌드/설치 의존성 없음
- 스타일 완전 통제(브랜드 톤 유지)

### 4-2. 권장 대안(추후)
- `lucide-react` 같은 아이콘 라이브러리 도입
  - 장점: 아이콘 폭넓음, 유지보수 용이
  - 단점: 패키지 설치 필요

도입 시 기준:
- stroke 2px, 24px 그리드 기반
- 아이콘 크기 기본 `w-4 h-4`(버튼), `w-5 h-5`(CTA), `w-6 h-6`(섹션 토글)

## 8) 작업 결과 자체 평가(2025-12-23)

### 8-1. 모바일 사용성
- **하단 CTA 고정**: `/learn/step/[stepId]`에서 다음/이전이 하단 고정 + safe-area 대응으로 **항상 노출**됨.
- **표 가독성**: 정산 수수료 표가 모바일에서 **행 카드 구조로 변환**되어 가로 스크롤/축소 없이 읽힘.
- **터치 타겟**: 주요 CTA 버튼은 기본 `Button` 컴포넌트의 패딩/높이로 **44px 이상**을 만족하도록 유지.

### 8-2. 시각적 전문성(아이콘/일관성)
- **아이콘 통일**: 화살표/체크/잠금/FAQ 토글 등을 **선형 SVG**로 통일해 화면별 “이모지/문자 심볼 혼재”를 크게 감소시킴.
- **예외 허용**: 🎯/🎉 등 일부 이모지는 “콘텐츠 강조/감정” 역할로 남겨, 정보 아이콘(내비/상태)과 분리함.

### 8-3. 전환/이탈 관점
- `/qualification → /learn`에서 CTA 버튼이 텍스트+아이콘(→) 조합으로 **명확해졌고**, 학습 정보 요약은 모바일에서 **1열로 재배치**되어 읽기 쉬움.
- `/learn → /step → next` 플로우에서 다음 액션이 하단 고정되어 **진행 흐름이 끊기지 않음**.

### 8-4. 남은 개선 여지(추가 제안)
- `lucide-react` 등 외부 아이콘 도입 시, 현재 `icons.tsx`를 교체/매핑하여 **아이콘 종류 확장** 가능(설치 환경이 정상화되면 권장).
- `/qualification/2026-waitlist`, `/qualification/no-business`의 상단/배지 이모지(📅/📋/💡 등)는 “정보 아이콘” 성격이므로 필요 시 **SVG로 추가 교체**하면 더 일관된 톤이 됨.

## 9) “AI 템플릿 느낌” 탈피를 위한 최적 방안(주제별 제안)

아이콘/모바일 최적화만으로도 품질이 올라가지만, “템플릿 감”을 지우고 **개성·전문성·완성도**를 확보하려면 아래 3축이 필요합니다.

- **A. 브랜드 자산을 ‘장식’이 아니라 ‘시스템’으로 사용**
- **B. 상호작용(마이크로 인터랙션) 품질 강화**
- **C. 정보 구조(IA) + 컴포넌트 라이브러리로 일관성 확보**

### 9-1. 브랜드 에셋 적극 활용(권장: 즉시 적용 가능)
- **브랜드 아이콘 시스템**: `BrandIcon`(브랜드 PNG) + `icons.tsx`(선형 SVG)로 역할 분리
  - **UI 아이콘**(화살표/체크/잠금/토글): 선형 SVG
  - **콘텐츠/감정 아이콘**(선물/배송/문구/좋아요 등): 브랜드 PNG
- **브랜드 패턴/라인 아트 사용 규칙**
  - 섹션 배경: `pattern01~03.png`를 `opacity 0.03~0.08`로 사용
  - 카드 구분선: `line_0x.png`를 섹션 분리용으로 제한적으로 사용
- **히어로/완료 화면의 “한 장면” 만들기**
  - 일러스트 1개(예: 선물/좋아요) + 카피 1줄 + CTA 1개를 “주인공”으로 두고 나머지는 보조

### 9-2. UI 프리미티브/컴포넌트 라이브러리 도입(권장)
“AI 템플릿”처럼 보이는 큰 이유는 **상호작용(열림/닫힘/포커스/키보드)**과 **디테일(애니메이션/간격/상태)**이 빈약하기 때문입니다.

추천 조합(프론트):
- **Radix UI + shadcn/ui**
  - 도입 목적: **접근성 보장 + 고급스러운 인터랙션**
  - 우선 적용 컴포넌트:
    - FAQ: Accordion
    - 안내/주의: Alert
    - 진행 안내: Tabs
    - 퀴즈 결과/제출: Dialog
    - 링크/버튼 보조: Tooltip
    - 완료/저장 피드백: Toast(sonner 등)

도입 순서(현 코드 영향 최소화):

- **폼 고도화(react-hook-form + zod)**: 입력/검증/에러 포커스 이동까지 완성도 상승 여지가 큼(패키지 추가 필요).
- **Tooltip**: “왜 필요한가요?” 같은 도움말을 작은 아이콘/툴팁으로 분리하면 정보 밀도가 더 좋아짐.

## 13) 세부 페이지 아이콘/컴포넌트 리뉴얼(반복 아이콘 문제 해결)

### 13-1. 문제(대표 예: 판매 가능/불가능 품목)
- 동일한 ✅ 아이콘이 여러 섹션에서 반복되면 “정보의 의미”가 아이콘으로 구분되지 않아 **전문성이 떨어져 보임**
- 텍스트 제목에 이모지가 포함되면, 레이아웃이 흔들리고 브랜드 톤이 약해짐

### 13-2. 해결 전략(의미 기반 아이콘 토큰)
- 섹션 아이콘을 “이모지”가 아닌 **의미 토큰**으로 정의:
  - `ok` / `no` / `warn` / `info`
  - 필요 시 `brand:<name>`로 브랜드 PNG 사용
- 렌더링은 `SectionIcon`이 담당:
  - 토큰 → `lucide-react` 아이콘(색/배경 포함)
  - 브랜드 토큰 → `BrandIcon`

적용 파일:
- `src/components/learning/SectionIcon.tsx` (신규)
- `src/components/learning/InfoGraphic.tsx`
- `src/app/learn/step/[stepId]/page.tsx`
- `src/data/contents.ts` (예: 판매 가능/불가능/MSDS 섹션 토큰 적용)

## 16) 작업 경과 자체 진단(현 시점) & 다음 작업 제안

### 16-1. 자체 진단 체크(전/후)

- **아이콘 중복(대표 스크린샷 이슈)**  
  - 전: ✅/❌/⚠️ 등 이모지 기반 아이콘이 여러 섹션에서 반복되어 “의미 구분”이 약함  
  - 후: `ok/no/warn/info` 토큰 + `SectionIcon`으로 의미 기반 아이콘을 렌더링 → **반복 아이콘 문제 구조적 해결**

- **관련 링크 아이콘(세부 페이지 완성도)**  
  - 전: 링크 리스트가 항상 동일 아이콘(고정)으로 보여 “리스트의 성격”이 구분되지 않음  
  - 후: `ExternalLinkItem` 도입으로 `link.icon` 토큰 기반 렌더링 → **링크 리스트도 같은 규칙으로 통일**

- **폼 입력 UX(전화번호/도움말/실패 피드백)**  
  - 전: 전화번호 포맷이 입력마다 제각각, “왜 필요한지” 설명이 텍스트로만 있어 집중을 깨뜨림  
  - 후: 전화번호 자동 하이픈 포맷 + Tooltip(모바일 클릭/데스크톱 호버/포커스) + 토스트/포커스 이동 → **입력 완성도 개선**

### 16-2. 남은 리스크(중요)
- `package.json`에 `react-hook-form`, `zod`, `@hookform/resolvers`를 추가했으므로  
  **`package-lock.json` 갱신이 필요**합니다.
