# 글로벌 작가 온보딩 페이지 — 최종 점검 & 평가(기획 ↔ 개발 역할 분담)

## 0) 목적(한 줄)
**작가님이 “문의 없이도” 글로벌 작가 등록까지 자연스럽게 완료하도록, 모바일 기준 가독성과 정보 위계를 최적화한 온보딩 플로우를 제공해요.**

---

## 1) 역할 분담(기획 vs 개발)

### 1-1. 기획(UX/콘텐츠) 역할
- **전환 목표 정의**: 최종 목적을 “글로벌 작가 등록” CTA로 고정
- **정보 위계 설계**: 한 화면에서 읽히는 핵심은 2~3개, 나머지는 추가 정보(아코디언)
- **불필요한 문의 유도 제거**: ChannelTalk 유도/버튼/문구 제거, FAQ/부록으로 대체
- **카피 톤**: idus 톤앤매너에 맞춘 존댓말/구어체(~해요/할 수 있어요) + 과한 명령형 최소화

### 1-2. 개발(구현/안정성) 역할
- **모바일 레이아웃 안정화**: 한글 줄바꿈(keep-all) + 긴 문자열(email/제목) 예외 처리
- **아이콘 시스템 정리**: emoji 난립 제거, semantic/brand 아이콘으로 일관화
- **학습 화면 구성**: 핵심 섹션 vs 추가 정보(아코디언) 자동 분류/수동 지정 병행
- **데이터 아카이빙 경로**: Railway 백엔드 우선, 실패 시 폴백 정책을 제어 가능하게 정리

---

## 2) 최종 구현 요약(핵심 개선점)

### 2-1. “핵심 vs 추가 정보” 정보 위계
- **Step 상세 화면**
  - 핵심(최대 3개) → 바로 노출
  - 부가 정보 → “추가 정보” 아코디언으로 노출 강도 하향
  - `importance: 'secondary'` 또는 `TIP/안내/참고/추가` 자동 분류로 아코디언 일관성 유지
- **이번 최종 반영**
  - 판매 가능/불가능: `MSDS 제출 필요 품목`을 추가 정보(secondary)로 이동
  - 번역 가이드: `번역 팁`을 추가 정보(secondary)로 이동

### 2-2. 모바일 가독성(줄바꿈) 품질
- 전역: 한글 단어 단위 줄바꿈을 기본으로 적용
- 예외: 이메일/메일제목처럼 긴 문자열은 `break-all`로 분절되어 화면 밖으로 튀지 않게 처리
- 타이틀: 필요 영역에 `text-balance`를 적용해 어색한 줄바꿈을 완화

### 2-3. 아이콘/컴포넌트 일관성
- Section 아이콘은 항상 “박스 안에” 보이도록 통일(emoji/국기 포함)
- 링크 영역은 truncate(…)를 제거해, 모바일에서도 제목/설명 전체를 읽을 수 있게 처리

### 2-4. 완료(축하) 페이지의 목적 집중
- 학습 결과/다음 단계 등 “행동에 직접 도움이 덜 되는 요소”는 제거
- 최종 목적: **글로벌 작가 등록 CTA**에 집중 + 부록/FAQ로 재접근 경로 제공

## 3) 사용자 관점(입점 작가 “빙의” 점검 체크리스트)

### 3-1. “내가 뭘 하면 되지?”가 빠르게 해결되는가
- 핵심 행동이 상단에서 1~2번 읽기만 해도 이해돼요
- 하위 디테일(MSDS/번역 팁/안내)은 “추가 정보”로 내려가 피로도를 줄여요

### 3-2. “돈/리스크” 정보가 과하게 튀지 않는가
- 수수료 정보는 확인 가능하되, 과도하게 강조되지 않게 노출 강도를 낮춰요(아코디언/작은 폰트)

### 3-3. “문의 없이도” 진행 가능한가
- FAQ/부록을 통해 필요한 정보를 스스로 찾을 수 있어요
- 링크가 잘려서(…) 못 읽는 문제를 제거해 이탈/문의 포인트를 줄였어요

### 3-4. 모바일에서 스트레스 없이 읽히는가
- 제목/문장 줄바꿈이 자연스럽고, 버튼/표/긴 텍스트가 화면 밖으로 튀지 않아요
- 터치 인터랙션(툴팁/복사 버튼)이 안정적으로 동작해요

## 4) 남은 리스크/운영 체크(권장)
- **데이터 아카이빙(Railway → Google Sheets)**는 반드시
  - Railway에 `GOOGLE_SHEETS_*` 환경변수 세팅
  - 시트를 서비스 계정 이메일에 편집 공유
  - Vercel의 `RAILWAY_BACKEND_URL`에 `https://` 포함
  - 폴백 정책(`ALLOW_GOOGLE_SCRIPT_FALLBACK`)을 운영 정책에 맞게 선택

### 4-1) 배포 직전 “체크리스트”(권장)
- **UI/UX**
  - 모바일에서 제목/본문이 단어 단위로 자연스럽게 줄바꿈되는지(특히 긴 문자열/메일 주소)
  - 학습 상세에서 핵심 카드 2~3개 + 추가 정보(아코디언) 구조가 유지되는지
  - 링크 제목이 (…)로 잘리지 않고 전체를 읽을 수 있는지
- **데이터 저장**
  - Vercel: `RAILWAY_BACKEND_URL=https://...` 확인
  - Railway: `/health`에서 `GOOGLE_SHEETS_*`가 모두 `true`인지 확인
  - Google Sheets: 서비스 계정 이메일 편집 공유 확인
  - 중복행 방지 정책: `ALLOW_GOOGLE_SCRIPT_FALLBACK` 운영 정책에 맞게 설정(권장: 기본 비활성)

## 5) 자체 평가(점수는 “운영 기준”으로 해석)
- **전환(등록까지 도달)**: 높음 — 완료 페이지/CTA 집중, 불필요 요소 제거
- **참여 난이도(모바일 기준)**: 높음 — 줄바꿈/긴 문자열/아이콘/버튼 레이아웃 정리
- **콘텐츠 완결성**: 높음 — 물류/정산/등록/품목/MSDS/번역/운영 TIP/부록/FAQ 커버
- **논리 구조**: 높음 — 핵심→보충(아코디언) 구조로 피로감 감소
