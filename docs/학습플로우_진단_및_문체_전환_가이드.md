# 글로벌 작가 온보딩 — 학습 플로우 진단 & 문체(~해요) 전환 가이드

## 1) 현재 학습 플로우(작가 관점) 진단

### 1-1. 작가의 “입점 의사결정” 흐름과 현재 구성의 정합성
- **작가의 핵심 질문(순서)**  
  - **(A) 이거 하면 뭐가 좋아요?** → 글로벌 판매의 가치/혜택(idus가 해주는 것)  
  - **(B) 나는 뭘 해야 해요?** → 최소 행동(ON/발송/등록)  
  - **(C) 돈/리스크가 궁금해요** → 수수료/정산/부가세(영세율)  
  - **(D) 운영이 가능한지 불안해요** → 주문/문의/고객 특성 TIP  
  - **(E) 지금 당장 다음 행동은?** → 글로벌 작가 등록 + 작품 글로벌 판매 ON

- **현재 플로우 적합도(요약)**  
  - **장점**: STEP1에서 “idus 지원”을 먼저 보여주고, STEP2/3에서 실제 운영 포인트를 이어감 → 의사결정 구조가 맞음  
  - **보완**: 각 카드에서 “핵심/부가” 구분이 약하면 **정보량이 많아 보이며 피로감 증가** → “핵심 2~3개 + 나머지 접기(추가 정보)”가 안정적으로 유지되어야 함

### 1-2. 학습 과정에서 빠지면 안 되는 필수 정보 체크
- **글로벌 판매 전제/혜택**
  - 해외 배송/통관/검수/포장/idus 지원 범위
  - 작가가 해야 하는 최소 행동(글로벌 ON + 물류센터 발송)
- **물류**
  - 물류 흐름(영상) + 국내 물류센터 주소 + 포장/발송 유의
- **정산**
  - 수수료 구조(표) + 정산 주기/통화(원화) + 영세율(0%)
- **작품 등록**
  - 글로벌 ON + **영문(일본어)** 입력
  - 판매 가능/불가 품목 + MSDS 제출 필요 + 제출처/메일제목
- **운영**
  - 문의 대응 원칙(번역 포함) + 일본/영어권 고객 특성 + 공통 판매 TIP
- **문서**
  - 소포수령증 신청 방식(월초 메일 링크)

→ 위 항목은 현재 구성에 포함되어 있고, 다음 단계는 “**각 항목이 ‘핵심 카드’로 적절히 노출되는지**”를 지속 점검하는 것입니다.

### 1-3. 인지 피로(정보량) 관점에서의 개선 기준
- **한 화면에서 즉시 읽혀야 하는 것**: 핵심 카드 **2~3개**
- **있으면 좋은 보충 정보**: “추가 정보(아코디언)”로 다운그레이드
- **동급 주제는 동급 레벨로 노출**: 예) 일본 고객 특성 ↔ 영어권 고객 특성은 같은 레벨(둘 다 핵심 카드)

## 2) “추가 정보(아코디언)” 운영 규칙(권장)

### 2-1. 섹션 중요도 표기
- `src/data/contents.ts`의 섹션에 다음 값을 사용:
  - `importance: 'secondary'` → 부가 정보로 간주(아코디언으로 내려감)
  - `highlight: true` → 핵심 강조(핵심 카드 우선 배치)

### 2-2. 데이터가 없어도 자동 분류되는 규칙(보조)
- 제목이 `TIP`, `안내`, `참고`, `추가`로 시작하면 자동으로 secondary로 취급

## 3) 문체(~해요) 전환 가이드 (idus 톤앤매너)

### 3-1. 기본 원칙
- **존댓말 구어체**를 기본으로 해요: “~해요 / ~할 수 있어요 / ~하면 돼요”
- **명령형 최소화**: “~하세요”는 꼭 필요한 CTA에서만 사용해요
- **불안을 줄이는 문장**: “걱정하지 않아도 돼요”, “idus가 도와드려요”
- **짧은 문장 + 포인트 먼저**: 결론 1줄 → 근거/설명

### 3-2. 권장 표현 예시
- “확인하세요” → “확인해요 / 확인하면 좋아요”
- “필수입니다” → “꼭 필요해요”
- “제출 바랍니다” → “미리 보내주면 돼요”
- “처리” → “진행 / 진행해요”

### 3-3. 적용 범위(우선순위)
- **1순위**: `src/data/contents.ts`의 모든 섹션/요약/설명 문구
- **2순위**: 주요 CTA(버튼/카드 카피): `/learn`, `/learn/step/*`, `/complete`, `/faq`
- **3순위**: 오류/안내 메시지(Toast/Modal)

### 3-4. 전환 작업 체크리스트
- 문장 끝을 “~합니다/하세요” → “~해요/돼요/할 수 있어요”로 일괄 전환
- ‘딱딱한 용어’(처리/진행/완료/등록) 중 불필요하게 강한 단어 완화
- 핵심 가치는 반복해도 됨: “idus가 도와드려요”, “작가님은 최소 행동만 하면 돼요”
